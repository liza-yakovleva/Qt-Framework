## 🎯 Огляд проєкту

Даний застосунок є частиною циклу лабораторних робіт з розробки інтерфейсів користувача. У цій ітерації реалізовано форму опитування для зібрання зворотного зв'язку від користувачів за допомогою **Python 3** та **PySide6** (офіційна реалізація Qt для Python).

Застосунок демонструє сучасні підходи до розробки GUI, включаючи розділення забезпечення (Separation of Concerns), валідацію даних та безпечну роботу з файлами.

---

## 🛠️ Технічні особливості

### High-Contrast UI
- Використано кастомні стилі **QSS** для забезпечення максимальної читабельності
- Обраховані кольори для максимальної контрастності тексту
- Адаптивність до різних системних тем (світла та темна)

### Layout Management
- Інтерфейс побудований на базі **QVBoxLayout** та **QHBoxLayout**
- Забезпечує адаптивність елементів при зміні розмірів вікна
- Логічна ієрархія компонентів

### Data Validation
- Реалізовано перевірку обов'язкових полів введення
- Валідація повніменів користувачів (мінімум 3 символи)
- Перевірка діапазону значень (вік від 16 до 100 років)
- Користувач отримує зрозумілі повідомлення про помилки

### Safe I/O Operations
- Збереження даних виконується через контекстні менеджери (`with open`)
- Кодування **UTF-8** для запобігання втрати даних та витоків пам'яті
- Обробка помилок файлової системи

## 🏗️ Архітектура

Проєкт реалізовано в рамках **об'єктно-орієнтованої парадигми (ООП)** з дотриманням SOLID принципів. Весь графічний інтерфейс описаний декларативно у коді, що є сучасним стандартом для гнучкої розробки динамічних GUI.

### Принципи організації

✅ **Розділення забезпечення (SoC)**
- Логіка, стилі та структури даних розділені по різним модулям
- Кожен модуль має одну та тільки одну відповідальність

✅ **DRY (Don't Repeat Yourself)**
- Уникнено дублювання коду
- Конфігурація винесена в окремий модуль

✅ **Конфігурованість**
- Всі константи зберігаються в `config.py`
- Легко змінювати параметри без редагування основного коду

✅ **Масштабованість**
- Додавання нових функцій не потребує змін в існуючому коді
- Модульна структура дозволяє легко розширювати функціональність

✅ **Тестованість**
- Кожен модуль можна тестувати окремо
- Залежності чітко визначені та видні

✅ **Читаність та підтримуваність**
- Понятні імена функцій та змінних
- Рясна документація через docstrings та коментарі

## 📖 Опис модулів

### `main.py` — Точка входу
```python
class SurveyApplication(QMainWindow)
```
- Головний клас застосунку
- Управління вікном та його ініціалізацією
- Координація роботи всіх інших модулів
- Функція `main()` для запуску застосунку

### `config.py` — Константи та конфігурація
Містить всі налаштування проекту:
- Параметри вікна (розмір, назва, заголовок)
- Діапазони значень (вік: 16-100, рейтинг: 1-5)
- Список технологій для опитування
- Налаштування файлів та кодування
- Шрифти та розміри

### `styles.py` — QSS стилізування
- Вся таблиця стилів QSS в одному місці
- Забезпечує єдиний візуальний стиль застосунку
- Легко змінювати дизайн та кольори без змін коду

### `models.py` — Структури даних
```python
@dataclass
class SurveyData
```
- Зберігає структурований представлення даних анкети
- Методи для форматування та серіалізації даних
- Автоматичне встановлення часової мітки

### `ui.py` — Побудова інтерфейсу
```python
class UIBuilder
```
- Статичні методи для створення кожного компонента UI
- Функція `build_main_layout()` для компонування елементів
- Централізоване управління структурою інтерфейсу

### `handlers.py` — Обробка подій
```python
class SurveyEventHandler
```
- Обробка натисків кнопок та користувацьких взаємодій
- Збір даних з форми
- Показ діалогів (помилок, успіху)
- Очищення форми після успішного збереження

### `utils.py` — Утилітарні функції
Функції валідації та I/O:
- `validate_user_name()` — перевірка імені
- `validate_age()` — перевірка віку
- `validate_rating()` — перевірка оцінки
- `save_survey_to_file()` — збереження у файл
- `format_technologies()` — форматування списку технологій

## 📋 Функціональність

Застосунок реалізує опитування користувача з такими питаннями:

1. **Прізвище та ім'я користувача** (текстовий ввід)
   - Обов'язкове поле
   - Валідація: мінімум 3 символи

2. **Вік користувача** (числовий spinner)
   - Діапазон: 16-100 років
   - Значення за замовчуванням: 21

3. **Оцінка складності проекту** (радіокнопки 1-5)
   - Обов'язковий вибір
   - Значення за замовчуванням: 5

4. **Обрання технологій** (чекбокси)
   - Python 3 & PySide6 (Qt)
   - Система контролю версій Git
   - Принципи дизайну інтерфейсів

5. **Додаткові пропозиції** (багаторядковий ввід)
   - Необов'язкове поле
   - Для додаткових коментарів

## 🔍 Приклад використання API

### Додавання нового питання:

1. Додайте константу в `config.py`
2. Створіть метод в `ui.py` (класс `UIBuilder`)
3. Додайте обробку в `handlers.py`
4. Обновіть модель даних в `models.py` (клас `SurveyData`)

### Змінення дизайну:

Просто редагуйте `styles.py` — жоден інший файл не потребує змін!

### Додавання нової валідації:

Додайте функцію в `utils.py` та використайте її в `handlers.py`

## 🐛 Вирішення проблем

**Помилка: "No module named 'PySide6'"**
```bash
pip install PySide6
```

**Помилка: "No module named 'config'"**
- Переконайтесь, що всі файли розташовані в одній папці
- Запустіть командою: `python main.py`

**Проблеми з кодуванням (подивкові символи)**
```bash
python -Xutf8 main.py
```
